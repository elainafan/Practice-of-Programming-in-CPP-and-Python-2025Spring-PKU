## 前言
此repo是北京大学《程序设计实习》课程的相关资料，作者于2025年春季学期选修了该门课程，授课教师为张勤健老师。

## 课程概况
供参考的课程分值分布：平时30%+期中35%+期末35%

其中平时分中，有5%为魔兽世界大作业，10%为QT/PYQT大作业，其余是作业+上机考勤分。

期中考试的形式为12道C++代码填空题+20道选择题，代码填空非线性给分（根据每年的完成情况跟难度，应该是变化的），占期中分数的80%，20道选择题，每道占期中分数的1%，一共20%。

期末考试的形式是12道Python代码题（注意，此处仅涉及Python面向对象部分是代码填空），亦为会变化的非线性给分。

课程内容大致可以分为C++面向对象和Python基础两部分，期中考试为前半部分，涉及内容大致为类与对象、运算符重载、继承与多态、模板、STL、C++11高级特性，期末考试为后半部分，但基础内容（也就是Python(I)(II)(III)(IV)会在期中前教授，期中后教授pandas、bs4、twinker等内容，期中后只有前两者为考点），大致可以分为Python基础语法、lambda表达式、Python类与对象、闭包生成器等等特性和pandas、bs4等库应用。

课程从第二周或者第三周开始会组织周末上机，分为四个时段，若在本班规定上机时段抽不出时间，可以跟两班助教申请调换到其他班的上机时间，或者直接缺勤失去考勤分数，上机题目后续也会放出供同学们补题、提交）。

课程的Bonus有PKUCPC和路演加分，PKUCPC对于非OI选手而言基本只能拿到做出签到题的一分（总不可能连签到题都做不出来吧！就是d2A-d2B难度），被选上路演可以拿到2-3分（大概？因为作者没有被选上，此处为道听途说的）的加分，见下文``QT大作业``部分。

## 学习&&考试复习相关
首先，如果想熟练掌握课程内容并且在机考中获得满意的分数的话，一定要认真理解课件，并且认真完成机考和平时练习习题，~~条件允许的话也可以像作者一样写博文~~。

深入到具体内容，个人认为C++面向对象这部分，它的题目可能会涉及多个知识点，难点大概是模板跟STL（前者重点考察理解，后者有点类似简单OI题），偏点则是C++11高级特性中除lambda表达式以外的几个内容，当然因为期中考有选择题，这会促使同学进行地毯式复习。

而对于Python基础这部分，由于平时上机只涉及Python(I)(II)这两部分，而考试对Python(III)(IV)这两部分的考察也很重，因此需要重点复习课件和相关作业，而pandas和bs4涉及的函数等等很多，考试会提供材料，前面熟悉的话现学时间很充裕，而Python(I)(II)基本是简单算法（最深到简单DFS）部分，如果不放心随便找点题（像作者是找的Openjudge上的程设Python Mooc）训练即可。

期中前部分一定要细致弄懂，作者翻新博文的时候才发现很多部分当时也一知半解，~~真不知道机考怎么把题做出来的~~。

### 关于魔兽世界大作业？
作者是在刚开学的三天内就先写完了这四部分，因此没有涉及任何类与对象相关知识，仅供参考。

如果你也有提前做的相关需求，可以在[程设Mooc](http://cxsjsxmooc2.openjudge.cn/warcraft/)提交进行测试。

同时，按照正常教学进度，老师在每个部分会下发对应的测试数据，便于Debug，如果你也有提前做的相关需求，可以前往``CSDN``的某篇文章下载测试数据，由于时间过于久远作者找不到网址了，或者找你的助教要，需要注意的是其中``魔兽世界4：终极``的测试数据中，疑似存在三个数据四舍五入精度有点问题，作者采用特判将其通过。

### 关于大作业？
大作业的十分，个人感觉只要不划水，前端跟功能稍微完善一下，就相对好拿满。

首先要说明的是，与2024春之前的要求不同，需要实现的为更贴近日常生活的应用，而游戏类应用并不提倡，由2-3位同学（一般是3位同学）使用Github协作完成，于期末考后（大概七月初）提交给助教，包括源码和演示视频，25春内容改革后开始可以使用PYQT以及接入AI相关API。

当然，在大概五月底，如果已经完成大致功能，可以向助教提交进入路演的申请（只需提交演示视频），会有一节课专门进行路演，如果通过审核则可获得一定加分（道听途说为总评2-3分），这里展示一位同学通过路演的项目链接。
